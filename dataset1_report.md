> 同 dataset2 报告一样，专业词全用生活例子解释。
> 重要更正：竞赛真实 MRR — **dataset1 = 0.44，dataset2 = 0.68**。

## 一、这个数据集是什么？

想象一个**社交平台**（微博、Twitter、Reddit 都可能）：

- 每个人既可以发消息**（src / 发送者）**，也可以收到别人的消息**（dst / 接收者）**
- 同一个人会同时出现在两边 ← 这是 dataset1 和 dataset2 最大的区别
- **一条记录 = 某人在某秒向某人发了一条消息（或点赞、评论、关注……）**

训练数据共有约 **69 万条交互记录**，测试要求猜出 6 万多条查询中"下一个互动对象"是谁。

| 项目 | Dataset1（社交网络）| Dataset2（书店）|
|------|-------------------|----------------|
| 图类型 | **同构图**（人→人）| 二部图（用户→书）|
| 节点总数 | 23,852 | 63,348 |
| 训练边数 | 690,848 | 2,261,283 |
| 时间粒度 | **秒级** | 日级 |
| 重复交互比例 | **72.6%** | 2.2% |
| 候选中冷启动 | **3.5%**（极少）| 53.6% |
| 竞赛真实 MRR | **0.44** | **0.68** |

## 二、最大区别：同一个人既是发送者又是接收者

在 dataset2（书店）里，用户和书是两种不同的东西，用户只能买书，书只能被买。

在 dataset1 里，**人既能发消息也能收消息**：

```
96.2% 的"发送者"也曾经是"接收者"
92.4% 的"接收者"也曾经是"发送者"
```

就像微信里：你可以给别人发消息，别人也可以给你发消息，你们都在同一个联系人列表里。

这在图论里叫**同构图（Homogeneous Graph）**——所有节点是同一类东西，不像二部图那样有明确的"用户"和"物品"之分。

---

## 三、最显著特征：72.6% 都是"老熟人"重复联系

dataset1 里 **72.6% 的交互是重复边**——同一对人之间反复互动。

| 同一对人之间的交互次数 | 边的数量 |
|---------------------|---------|
| 1次（只联系过一次） | 89,614 对 |
| 2次 | 36,848 对 |
| 3次 | 18,684 对 |
| 4次 | 11,030 对 |
| 5次 | 6,899 对 |
| 10次及以上 | 有 6.8% 的边 |

**最高纪录：同一对人之间联系了 1,000+ 次！**

这完全不像买书（很少买同一本书两次），更像是**微信好友**——你可能每天都在和固定的几个人聊天。

---

## 四、冷启动问题很小（与 dataset2 完全相反）

| 项目 | dataset1 | dataset2 |
|------|---------|---------|
| 候选中从未出现过的节点 | **3.5%**（极少）| 53.6%（超过一半）|
| 测试查询中的候选全部认识 | 2.8% 的查询 | 0% |
| 每条查询平均已知候选数 | **96.5 / 100** | 46.4 / 100 |

也就是说，测试时给出的 100 个候选里，**几乎全是训练里见过的节点**，模型不需要猜"陌生人"，而是要从一堆"熟人"里猜对那个人。

这听起来更容易，但实际竞赛 MRR 反而**更低（0.44 vs 0.68）**——为什么？见第七节。

## 五、用户活跃度差异很大

| 指标 | 数值 |
|------|------|
| 用户（src）总数 | 22,093 |
| 平均每人互动次数 | 31.3 次 |
| 中位数（一半以上的人不超过）| **12 次** |
| 最活跃用户 | **1,333 次** |
| 只互动过 1-2 次的人 | 9.2%（冷清用户）|

比 dataset2（书店）的互动少很多（书店用户中位数也是14，但均值高达178），说明 dataset1 是个更"分散"的社交网络，大部分人只和少数几个人互动。

## 六、时间特征：秒级精度，间隔较短

```
训练时间范围：约 4 年（时间戳从0开始计秒）
测试时间范围：训练结束后紧接着的一段时间
时间粒度：精确到秒（dataset2 是按天）
```

**用户最后一次训练互动 → 测试查询的时间间隔**：

| 区间 | 查询数量 |
|------|---------|
| 不到 1 天 | 131 |
| 1-7 天 | 1,921 |
| 1-4 周 | **14,587** |
| 1-3 个月 | **33,760**（最多）|
| 3-6 个月 | 7,095 |
| 6-12 个月 | 2,528 |
| 1 年以上 | 681 |

平均间隔 **64 天**（约 2 个月），比 dataset2 的 220 天短得多，说明这个平台互动节奏更快。

## 七、为什么 MRR 只有 0.44？（比 dataset2 的 0.68 还低）

这是个看似矛盾的现象：候选几乎全是熟面孔（96.5% 已知），但猜对率反而更低。

**原因分析：**

### 原因1：候选全是"同等竞争者"
dataset2 里，已知物品 vs 神秘书 → AI 能用训练里的信息区分一部分。
dataset1 里，**100 个候选几乎全是训练中出现过的节点**，每个人都有历史记录，AI 必须靠更细腻的偏好判断来区分，难度更高。

### 原因2：历史重复互动混淆了方向
65.2% 的测试查询里，候选里有该用户历史上互动过的节点（平均 0.69 个）。
AI 可能会被历史互动迷惑：「这 100 个人里，我历史上联系过的有好几个，到底下一个是谁？」

### 原因3：训练数据量少
dataset1 只有 69 万条训练记录（dataset2 有 226 万条），而且节点总数相对差不多，每个节点平均只有 31 次历史可以参考，信息量有限。

### 原因4：自环（自己联系自己）
2.62% 的训练边是"自环"（发送者=接收者），1.3% 的测试候选里包含 src 自己。这种情况在书店场景里不存在，对模型是额外干扰。

---

## 八、AI 模型的预测分布

与 dataset2 不同，dataset1 的 AI **得分分布很宽**（标准差 0.27，dataset2 仅 0.056）：

- 大约 **一半** 的预测集中在 0.5 左右（模型很不确定）
- 但有相当比例的预测分布在 0～0.3（低分，自信"不是它"）和 0.7～1.0（高分，自信"就是它"）
- 说明 AI 对部分查询能给出有把握的判断，对另一部分则完全没把握

这比 dataset2（几乎全在 0.5 左右，完全随机）要好，也解释了为什么 MRR=0.44 而不是接近随机的 0.01。

## 九、可以怎么改进？

### 改进1：利用重复交互历史（最有价值）
72.6% 是重复边，说明**用户倾向于和固定的少数人互动**。
可以专门建一个"用户的常用联系人列表"，在 100 个候选里优先给历史联系过的人打高分。

### 改进2：图神经网络（GNN）——利用朋友的朋友
在社交网络里，"你的朋友的朋友"很可能成为你的下一个互动对象。
图神经网络能通过传递邻居信息，比简单的序列模型更好地捕捉这种结构。

### 改进3：时序衰减——越近越重要
间隔 1 周的互动比间隔 6 个月的更值得参考。加入时间权重，让近期互动的影响更大。

### 改进4：对称性利用——双向关系
如果 A 频繁联系 B，那 B 联系 A 的可能性也更高。可以同时考虑"谁联系过 src"和"src 联系过谁"两个方向。

### 改进5：共同邻居 / ItemCF 协同过滤
和我互动模式相似的人，他们联系的节点也可能是我的候选。
这是经典协同过滤的思路，在社交网络里效果通常很好。

## 十、总结对比

| | Dataset1（社交网络）| Dataset2（书店）|
|--|---------------------|----------------|
| **核心难题** | 熟面孔中猜对一个 | 超过一半答案是没见过的 |
| **主要特征** | 重复互动72.6%，同构图 | 二部图，每次买新书 |
| **冷启动** | 几乎没有（3.5%）| 非常严重（53.6%）|
| **AI现状** | MRR=0.44，有区分度 | MRR=0.68，表现更好 |
| **最佳改进方向** | 利用重复互动 + GNN | 冷启动物品表示 + 去偏 |
