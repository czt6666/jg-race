# 算法设计方案（理论最优，忽略实施难度）

---

## 一、是否需要分开设计？结论：**必须分开**

两个数据集的核心挑战完全不同，共用一套方案会在两边都留下明显性能空间：

| 维度 | Dataset1（社交图） | Dataset2（电商图） |
|------|-------------------|-------------------|
| 图类型 | 同构图（人→人） | 二部图（用户→商品） |
| 核心难题 | 72.6% 重复边，熟人堆里精确排序 | 54% 冷启动候选，区分已知/未知商品 |
| 冷启动压力 | 几乎没有（3.5%） | 严重（54%） |
| 序列长度 | 中位数 12，短 | 中位数 14，但均值 177，极长尾 |
| 时间间隔 | 中位数 48 天，近 | 中位数 187 天，远 |
| 关键信号 | 历史联系频率 + 时间衰减 | 购买序列 + 商品图结构 |
| 最优架构族 | 时序图网络（TGN 族） | 序列推荐（SASRec 族）+ 图归纳嵌入 |

---

## 二、Dataset1 最优方案

### 2.1 核心洞察

72.6% 的测试正样本很可能就是用户历史上频繁联系的"老熟人"。
最强的理论模型需要同时做到：
- **记住每对节点的历史互动积累**（频率 + 时间衰减）
- **捕捉图结构的"朋友圈"传播**（朋友的朋友也是候选）
- **序列感知**（最近几步的互动模式最重要）
- **100-way 精确重排序**（给定 100 个候选，用 cross-attention 做精排）

### 2.2 各层算法排序

**图结构层（最关键）**

| 排序 | 算法 | 理由 |
|------|------|------|
| 1 | **TGN (Temporal Graph Network)** | Memory 模块天然积累重复互动信号；图注意力聚合邻居；专为时序链接预测设计；在 Wikipedia/Reddit/ogbl-collab 等社交图 benchmark 排名第一 |
| 2 | **CAWN (Causal Anonymous Walks)** | 因果匿名随机游走，捕捉子图结构模式，对社交网络极强 |
| 3 | **DyGFormer** | 已有实现；Transformer over 时序邻居，长序列友好 |
| 4 | **GraphMixer** | 简洁 MLP-Mixer 架构，实测接近 TGN，训练更稳定 |
| 5 | **TGAT** | 时间编码 + 图注意力，TGN 的前身 |

**时序序列层**

| 排序 | 算法 | 理由 |
|------|------|------|
| 1 | **TiSASRec** | 在 SASRec 基础上加时间间隔编码，捕捉"多久联系一次"的节律 |
| 2 | **SASRec** | 当前 baseline，稳定 |
| 3 | **BERT4Rec** | 双向注意力提取更丰富的历史模式（适合 representation，不适合直接预测） |

**重复互动显式特征层**

| 特征 | 公式 | 说明 |
|------|------|------|
| 频率特征 | `freq(u,v) / total_interactions(u)` | 历史联系次数归一化 |
| 时间衰减特征 | `freq(u,v) × exp(-λ × Δt_last)` | 最近一次联系时间衰减 |
| 互惠特征 | `freq(v,u) / total_interactions(v)` | v 是否也频繁联系 u（双向） |

**精排层（最终 100-way 重排）**

| 排序 | 算法 | 理由 |
|------|------|------|
| 1 | **Cross-Encoder（全注意力精排）** | 把用户历史序列和每个候选节点拼接，用 Transformer 全注意力打分；比双塔 dot-product 强得多，100 候选规模完全可以跑 |
| 2 | **WARP 排序层** | 在精排头使用 WARP loss 直接优化 MRR |
| 3 | **LambdaMRR** | 直接对 MRR 梯度求导优化 |

**损失函数**

| 排序 | 损失 | 理由 |
|------|------|------|
| 1 | **1-vs-99 Hard Negative Sampled Softmax** | 用实际测试候选池的 99 个负样本训练，loss 空间和评测完全对齐 |
| 2 | **BPR + Hard Negative 增强** | 稳定，易实现 |
| 3 | **In-batch NCE** | 当前已用，基线 |

### 2.3 Dataset1 最终组合

```
TGN (时序图记忆)
  ↓ 节点嵌入
TiSASRec (序列建模)
  ↓ 序列嵌入
频率+时间衰减显式特征 融合
  ↓ 特征向量
Cross-Encoder 精排（用户上下文 × 100个候选 全注意力）
  ↓
1-vs-99 Hard Negative Sampled Softmax
```

**各组件分工：**
- TGN：捕捉节点级的长期关系积累（memory） + 图结构的朋友圈传播
- TiSASRec：捕捉时序节律（联系的时间模式）
- 频率/衰减特征：直接建模"老熟人"信号，弥补神经网络在这方面的短板
- Cross-Encoder：100 候选精排，充分利用所有候选间的相对对比信息
- Hard Neg Loss：train/eval 分布对齐

**理论天花板预估：ds1 MRR 0.44 → 0.65+**（主要增量来自 TGN 的 memory 机制 + Cross-Encoder）

## 三、Dataset2 最优方案

### 3.1 核心洞察

54% 冷启动候选得分均匀 ≈0.5，正样本可能是已知商品（AI 能识别）也可能是冷商品（AI 无法识别）。
最强模型需要：
- **对已知正样本：给出远高于 0.5 的分数**（更精准的序列建模）
- **对冷正样本：从 ID 结构或图邻居推断出略高于其他冷商品的分数**
- **处理长序列**（均值 177，少数用户有 1000+ 购买记录）
- **Session 结构**（同一天内的多次购买是一个 session，有内部关联）

### 3.2 各层算法排序

**序列建模层（最关键）**

| 排序 | 算法 | 理由 |
|------|------|------|
| 1 | **TiSASRec** | 时间间隔编码；187 天的长间隔需要时间感知；next-item 预测任务天然适合自回归 |
| 2 | **SIM (Search-based Interest Model)** | 阿里巴巴方案，专门处理 1000+ 长序列；对每个候选商品，先从历史中检索 top-K 相关购买，再做短注意力；不丢失长期信号 |
| 3 | **BERT4Rec + Session Block** | 同一天内的购买做局部 self-attention（session），跨 session 做全局 attention；捕捉 49.7% 同日多购行为 |
| 4 | **ComiRec（多兴趣抽取）** | 电商用户有多种购买偏好（数码、家居、书籍），多向量表示比单向量强 |
| 5 | **GRU4Rec** | RNN 建模购买序列，捕捉连续演化 |

**图结构层（主要解决冷启动）**

| 排序 | 算法 | 理由 |
|------|------|------|
| 1 | **PinSage（归纳式图卷积）** | Pinterest 专为大规模二部图设计；random walk + 邻居聚合；归纳式：冷商品通过邻居获得嵌入；工业界最成熟的冷启动图方案 |
| 2 | **LightGCN** | 纯 GCN 协同过滤，用户→商品二部图上 L 层传播；简洁强效；训练完后所有商品都有 CF-enhanced embedding |
| 3 | **GraphSAGE** | 通用归纳式图采样聚合 |

**冷启动专项**

| 排序 | 方案 | 说明 |
|------|------|------|
| 1 | **ID 分解嵌入（Quotient-Remainder Trick）** | 商品 ID 拆成 `商 = ID // K` 和 `余 = ID % K`，两个小嵌入矩阵相加；冷商品从 ID 空间邻居借到语义，无需训练图 |
| 2 | **PinSage 归纳嵌入** | 冷商品通过购买图上的 2 跳邻居聚合获得嵌入 |
| 3 | **DropoutNet** | 训练时随机将 item embedding 清零，强迫模型学会"只靠上下文信息时如何推断" |
| 4 | **候选集共现初始化** | 测试时：同一查询的 100 个候选中，冷商品 = 已知候选嵌入的加权均值初始化 |

**长序列处理**

| 排序 | 方案 | 说明 |
|------|------|------|
| 1 | **SIM（检索式兴趣模型）** | 对每个候选商品，从历史 1000+ 中快速检索 top-50 相关购买，再做精细注意力；O(1) 检索而非 O(N²) 注意力 |
| 2 | **ETA（Efficient Target Attention）** | 局部敏感哈希加速长序列注意力 |
| 3 | **截断 + 层级 Session 注意力** | 保留最近 N 个 session，session 内 attention，session 间 attention（两级） |

**流行度去偏**

| 排序 | 方案 | 说明 |
|------|------|------|
| 1 | **可学习去偏（IPS + 倾向分）** | 用 inverse propensity scoring 纠正训练分布的热度偏差，理论最优 |
| 2 | **对数去偏** | `分数 -= α × log(热度+1)`，α=0.5，本地验证 MRR +8× |

**精排层**

同 Dataset1：Cross-Encoder 对 100 候选做全注意力精排

**损失函数**

| 排序 | 损失 | 理由 |
|------|------|------|
| 1 | **1-vs-99 Hard Negative Sampled Softmax** | 直接用测试候选池的 99 个负样本做训练，训练和评测完全对齐 |
| 2 | **Sampled Softmax (τ=0.05)** | 大量随机负样本 + 温度缩放，强基线 |
| 3 | **BPR + Hard Neg** | 稳健 pairwise |

### 3.3 Dataset2 最终组合

```
PinSage (二部图归纳嵌入，解决冷启动)
  ↓ 商品初始嵌入（冷热商品统一表示）
ID 分解嵌入 融合（兜底冷启动）
  ↓ 增强商品嵌入
TiSASRec + SIM 长序列检索
  ↓ 用户兴趣向量
Session Block（同日购买局部 attention）
  ↓
流行度去偏（IPS 倾向分 or 对数去偏）
  ↓
Cross-Encoder 精排（用户上下文 × 100个候选 全注意力）
  ↓
1-vs-99 Hard Negative Sampled Softmax
```

**各组件分工：**
- PinSage：为每个商品（包括冷商品）提供基于图结构的归纳嵌入
- ID 分解：为极端冷商品（图上也是孤立节点）提供 ID 空间的语义兜底
- TiSASRec：主序列模型，时间感知
- SIM：处理超长历史序列，防止近期信号被稀释
- Session Block：捕捉同一天批量购买的内部关联
- 去偏：消除模型对热门商品的系统性偏好
- Cross-Encoder：充分利用 100 候选的相对对比信息做最终精排
- Hard Neg Loss：训练空间对齐测试评测

**理论天花板预估：ds2 MRR 0.68 → 0.80+**（主要增量来自 PinSage 冷启动 + Cross-Encoder 精排）

## 四、方案对比总结

| 设计层 | Dataset1（社交图） | Dataset2（电商图） |
|--------|-------------------|-------------------|
| **核心骨架** | TGN（时序图记忆网络） | TiSASRec + SIM（时间感知序列 + 长序列检索） |
| **图结构** | TGN memory + 邻居图注意力 | PinSage 归纳式二部图嵌入 + LightGCN CF |
| **重复/Session** | 频率+时间衰减显式特征 | Session Block（同日多购） |
| **冷启动** | 几乎不需要（3.5%可忽略） | PinSage 归纳 + ID 分解 + DropoutNet |
| **长序列** | 不需要（中位数 12） | SIM 检索式兴趣 + 两级 Session-Global 注意力 |
| **去偏** | 无需（重复交互不存在热度偏差问题） | IPS 倾向分 + 对数去偏（α=0.5） |
| **精排** | Cross-Encoder（100候选全注意力） | Cross-Encoder（100候选全注意力） |
| **损失** | 1-vs-99 Hard Neg Sampled Softmax | 1-vs-99 Hard Neg Sampled Softmax |
| **当前 MRR** | 0.44 | 0.68 |
| **理论天花板** | **~0.65+** | **~0.80+** |

## 五、最优方案的关键创新点

### 对 Dataset1 最重要的单点改进（按增量排序）

1. **Cross-Encoder 精排**：把双塔 dot-product → 全注意力 cross-encoder，100候选规模完全可承受
2. **TGN memory 机制**：自然积累"老熟人"信号，比 SASRec 更适合 72.6% 重复边场景
3. **频率+衰减显式特征**：神经网络的结构性补充，对重复边场景极其有效
4. **1-vs-99 Hard Neg Loss**：训练/评测对齐，直接在竞赛候选池上优化

### 对 Dataset2 最重要的单点改进（按增量排序）

1. **PinSage 冷启动嵌入**：让 54% 冷候选有意义的分数差异（而非全部 ≈0.5 随机）
2. **Cross-Encoder 精排**：对已知正样本的精确排序能力大幅提升
3. **SIM 长序列检索**：均值 177 的购买历史现在全部有效利用，而非截断
4. **1-vs-99 Hard Neg Loss**：训练/评测对齐
