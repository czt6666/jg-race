# Dataset2 数据分析报告（大白话版）

> 真实竞赛 MRR（SASRec+WD）：**dataset1 = 0.44 / dataset2 = 0.68**
> 正样本被随机打散在 c1~c100 任意位置，仅竞赛服务器知道真实位置。

---

## 一、这是什么任务？

**任务名称：时序图中的未来链接预测（Temporal Link Prediction）**

> 赛题原话："本赛题聚焦时序图中的未来链接预测任务。给定用户、物品、作者、网页等实体在历史上的交互序列……"

用大白话说：

- 一张"时间图"里，节点是各种实体（用户、商品、作者、网页……）
- 边是"某时刻发生的一次互动"（买了书、点击了商品、给文章打了评分……）
- **任务**：给出一个用户节点和一个时刻，从 100 个候选商品里猜出"他下一个要买/互动的是哪个"

评分方式：**100-way MRR**——正确答案排在第几名，越靠前得分越高。

---

## 二、Dataset2 是什么数据？（推测）

根据数据特征高度匹配 **Amazon 商品评论/购买数据**：

| 特征 | 数值 | Amazon 数据对比 |
|------|------|----------------|
| 时间戳 | 2000-04-12 ~ 2011-01-29 | ✓ Amazon 数据覆盖此时期 |
| 时间精度 | **日级**（86400 秒间隔）| ✓ 评论按天记录 |
| 用户数 | 12,708 | ✓ 商品类别子集规模 |
| 商品数 | 50,640 | ✓ 商品类别子集规模 |
| 图结构 | 二部图（用户→商品）| ✓ 用户只买商品，商品只被买 |
| 重复购买 | 仅 2.2% | ✓ 评论很少重复 |
| 月度趋势 | 2007~2008 达峰后下滑 | ✓ 符合 Amazon 评论增长曲线 |

**最可能的原型**：Amazon 某品类（书籍/电子产品/家居）的历史购买/评论记录。

---

## 三、图结构：完美二部图（用户→商品）

```
用户（src）: 12,708 个    商品（dst）: 50,640 个
用户 ∩ 商品 = 0  →  Overlap = 0%  →  完美二部图
```

- 用户只能"买"商品，商品只能"被买"
- 测试用户（2,180 人）全部出现在训练集中 → **无用户冷启动**

---

## 四、最大挑战：候选中超过一半是"神秘商品"

每条测试查询给出 100 个候选商品，其中：
- **~46 个**是训练中出现过的商品（AI 认识）
- **~54 个**是训练中从未出现的"神秘商品"（AI 不认识）

这里要特别说明：**c1 列不是固定的正样本**，正确答案（正样本）被随机打散在 c1~c100 中的某一列，只有竞赛服务器知道真实位置。之前分析"c1 的冷启动率"是无效的，因为 c1 只是一个随机位置。

正确的理解是：**100 个候选整体上约有 54% 是冷启动商品**，正确答案可能在其中，也可能是已知商品。

---

## 五、候选集的精心设计（出题人的陷阱）

竞赛方构造候选集时特意让它很难：

- 所有候选商品的热度分布与正样本完全一致（比值 1.006×）
- 冷启动/已知商品的比例，与正样本的分布一致
- 测试候选频率：神秘商品均值 137.7 次，已知商品 140.6 次 → 几乎相同

**结论：靠商品热度排序毫无用处，MRR 仅 0.056（接近随机）。**

---

## 六、神秘商品如何影响 AI 模型？

SASRec 模型给商品打分的方式：每个商品都有一个"脸"（嵌入向量），用户历史记录和商品"脸"对比后打分。

- **已知商品**：训练中出现过，有训练好的"脸"，分数有意义
- **神秘商品**：从未出现，"脸"是随机初始化的，所有神秘商品分数都约等于 0.5

当正确答案是已知商品时：AI 打出的高分能让它从 54 个随机得 0.5 分的神秘商品中脱颖而出。  
当正确答案是神秘商品时：AI 无法区分它和其他神秘商品，只能靠运气。

**这就是为什么 dataset2（更多冷启动）反而比 dataset1（几乎无冷启动）MRR 更高**：
dataset2 的 54 个冷启动候选竞争力弱，帮助了已知正样本排名；
dataset1 的 96 个已知候选个个都有训练过的"脸"，竞争更激烈。

## 七、本地验证 vs 真实测试的差异

| | 本地验证（split-1）| 竞赛测试集 |
|--|--|--|
| 商品冷启动比例 | 0% | ~54% |
| 负样本构造方式 | 随机（简单）| 精心设计（热度匹配）|
| SASRec MRR | 0.525 | **0.68** |

本地验证分数（0.525）低于真实竞赛分数（0.68）。

原因分析：
- 本地验证的负样本是随机的，AI 容易区分
- 真实测试的负样本虽然精心构造，但 AI 在真实数据上的泛化能力可能比在验证集上更好
- 另外，背景任务测出了"去热度偏差后本地验证 MRR 可达 0.577"，说明还有提升空间

## 八、时间特征

```
训练集：2000-04-12 ~ 2011-01-29（11年）
测试集：2011-01-30 ~ 2012-01-13（1年）
时间精度：日级（每天一个时间戳）
```

- 用户最后一次训练购买 → 测试查询：平均 **220 天**，中位数 **187 天**
- 月度趋势：2007~2008 达峰，之后逐渐下滑
- 49.7% 的（用户, 日期）对有超过 1 次购买 → 同日批量购买行为普遍

## 十、改进方向

### 改进1：去热度偏差（立竿见影）
AI 倾向于给热门商品打高分，但热门不等于是正确答案。  
公式：`调整分数 = 原始分数 - 0.5 × log(商品热度 + 1)`  
本地验证效果：MRR 从 0.073 → **0.577**（提升 8 倍）

### 改进2：神秘商品表示（核心难点）
神秘商品需要借助间接信息获得有意义的分数：
- **ID 分解嵌入**：把商品编号拆成大类/中类/小类，共享嵌入参数，让神秘商品借助同 ID 区间的商品特征
- **测试候选共现图**：同一查询里出现的商品可能是同类，用邻居商品的嵌入平均值初始化神秘商品

### 改进3：修复本地验证（最紧迫）
当前验证集无冷启动，评估结果不能真实反映比赛性能。需要在验证集里也加入约 54% 的冷启动候选。

### 改进4：用考试题练习（Hard Negative 训练）
测试集里的 99 个负样本虽不知道哪个是正样本，但可以用作训练时的"难负样本"，让模型学会在同等难度的负样本中做区分。

### 改进5：序列建模优化
- 扩大序列长度（median=14，但 95th 百分位=967）
- 考虑同日批量购买的 Session 结构
- 对最近交互加更大时间权重

| 项目 | 数值 |
|------|------|
| 图类型 | 二部图（用户→商品），疑似 Amazon |
