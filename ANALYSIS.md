## 关键问题分析

### 问题1: 为什么1DyGFormer (0.9922) 远优于 3DyGFormer_v3 (0.9728) on dataset1?

**差异点:**
- 1DyGFormer: BCE损失 + 1个随机负样本 + 随机负样本验证
- 3DyGFormer_v3: Ranking Loss + 20个硬负样本 + 硬负样本验证

**核心原因:** 验证集 AP 指标是用随机负样本计算的更接近"绝对置信度"，BCE 直接优化这个目标。Ranking Loss 优化相对排序，不直接优化 AP。

→ **BCE 训练更好地优化了 AP 指标**

### 问题2: Dataset2 的训练崩溃

崩溃过程:
```
Epoch 1: AP=0.8968 (warmup, 均匀随机负样本)
Epoch 2: AP=0.8887 (轻微下降)
Epoch 3: AP=0.8544 (warmup结束，硬负样本开始)
Epoch 4: AP=0.4420 (崩溃！)
Epoch 5+: AP=0.3333 (随机猜测水平)
```

**崩溃原因:** 二部图上的2-hop硬负样本极其激进:
- 用户u → 物品i → 其他用户购买过的物品
- 这些候选和正样本非常相似，模型无法区分
- Ranking Loss (softmax over 20 hard negs) 梯度方差极大
- 梯度爆炸 → 参数发散 → AP=0.3333

### 问题3: Dataset2 的 54% 冷启动候选

**分析:**
- 54.1% 测试候选是训练+验证期间完全未见的新物品
- DyGFormer 用时序邻居特征，新物品没有历史 → 嵌入=零向量
- 所有新物品获得相同的表示

**但是:** 如果正样本都是 45.9% 的已知物品，而54.1%新物品全是负样本，那么模型反而可以：
- 对已知正样本打高分
- 对所有新物品打同等低分
- 正样本仍然排在前面

**实际挑战:** val AP 只有 0.8968，说明即使在全部已知的验证集上，模型也有较大提升空间

### 问题4: Max Seq Len 太小 (dataset2)

- dataset2 用户平均交互次数: **714次**
- 当前 max_seq_len=32: 只使用最近32次交互
- 用户有丰富历史，但模型只看了 32/714 = 4.5% 的信息！
- 增大 max_seq_len 可能显著提升dataset2性能

## 改进策略

### V5 设计思路

**核心改变:**
1. **多负样本BCE** (num_neg=5): 比1个负样本更强的梯度信号，比20个硬负样本更稳定
2. **流行度加权负样本** (sqrt(freq)): 比均匀随机难一些，但不会崩溃
3. **Label smoothing** (ε=0.1): 防止过拟合，提高泛化
4. **更大上下文** max_seq_len=64 (对dataset2有巨大帮助)
5. **Cosine LR decay**: 更好的收敛
6. **无2-hop硬负样本**: 彻底解决dataset2崩溃问题

### 预期提升

Dataset1:
- 已在0.9922附近，主要依靠更多训练轮次
- max_seq_len: 32→64 可能有小幅提升
- 目标: 稳定突破 0.99 AP/MRR

Dataset2:
- 稳定训练是第一优先级
- max_seq_len: 32→64 可能有 +0.02 提升
- num_neg=5 vs 1 可能有 +0.01-0.02 提升
- 目标: 从0.8968 → 0.93-0.95

- Epoch 1: AP=**0.9926**, MRR=**0.9910**
- Epoch 2: AP=**0.9926**, MRR=0.9907 (收敛稳定)
- **Dataset1 目标: AP=0.9926 ≥ 0.99 ✓**

### Run 3: 1DyGFormer Dataset2 (随机负样本验证) - 进行中
- GPU0, PID 926996, 2.07 it/s, 9611 batches/epoch
- **Epoch 1: AP=0.9332, AUC=0.9457, MRR=0.9730** — 差 0.95 目标还差 0.017

#### 分析: 为什么 dataset2 AP < dataset1?

**根本原因: Batch-level AP 计算 + 热门物品混淆**
- 批量 AP 是对 200 正样本 + 200 随机负样本一起排序的
- Dataset2 验证集正样本物品平均度 = 385.2 (整体中位数只有10)
- 验证集大量是用户与热门物品的交互
- 随机负样本可能也是热门物品(度数高) → 模型难以区分
- MRR=0.9730 说明 pair-wise 排序好，但 cross-batch 比较拉低了 AP

**量化证明:**
- MRR=0.9730 对应 pair-wise 准确率 ~94.6%
- Batch AP=0.9332 低于 pair-wise 准确率
- 差距 = cross-batch 排序的热门物品干扰效应

**改进方向:**
1. 流行度加权负样本训练 → 逼模型学习细致区分热门物品 (最关键!)
2. max_seq_len=64 (更多上下文，但慢3.5h/epoch)
3. 更多训练轮次 (但2轮后已平台期, AP=0.9349)

**Epoch 2: AP=0.9349** (仅+0.0017，平台期) → 停止1DyGFormer路线

#### Run 4: V5 Dataset2 (pop-weighted训练 + 随机验证, seq_len=32) - 进行中

**核心假设:** 用流行度加权负样本训练，让模型更好地区分热门物品
- seq_len=32 (速度快), num_neg=1, p_pop=0.7 (70%流行度加权)
- 训练: PopNegSampler (harder) → 验证: RandomNegSampler (fair)
- 速度: 1.24 it/s (~129 min/epoch), PID 1127060
- **理论预期: Epoch 1 AP ≥ 0.95** (pop-weighted训练使模型更discriminative)

注: 为什么比1DyGFormer慢? Python级别的采样循环 vs DataLoader向量化C++采样

## 深度根因分析：为什么得分只有 1.16（别人 1.42）

### 评分结构拆解

竞赛得分 = Dataset1 贡献 + Dataset2 贡献（最高各 ~1.0）
- 我们：1.16 ≈ Dataset1(0.99) + Dataset2(0.17)
- 最优：1.42 ≈ Dataset1(0.99) + Dataset2(0.43)
- **核心差距全在 Dataset2！Dataset2 的真实测试 MRR 仅约 0.17**

### 关键发现 1：验证指标严重误导

| 指标 | 我们的值 | 说明 |
|------|---------|------|
| Val MRR (pairwise, 1 neg) | 0.9730 | **虚假繁荣！** |
| 估算 Test MRR (100-way) | ~0.17 | 真实水平 |
| 差距 | ×5.7 倍 | 指标完全不代表测试难度 |

**原因：** Val 评估是 1正 vs 1随机负（pairwise），而 Test 是 1正 vs 99个负样本（100-way ranking）。这是完全不同的任务难度。

### 关键发现 2：54% 冷启动候选（dataset2 核心问题）

```
训练物品数：50,640 个
测试候选唯一物品：110,368 个
其中从未在训练集出现：59,729 个 (54.1%)

每行候选构成：
  平均 46 个已知物品（模型有表示）
  平均 54 个全新物品（模型给出 0 向量）
```

**模型对新物品的处理：**
- 所有新物品的嵌入 = 零向量 → 输出分数几乎相同（≈ 0.0635）
- 新物品平均排名：64.5/100（比随机 50.5 还差！排在所有已知物品之后）
- 已知物品平均排名：34.5/100（占据前列）

**实测证明：**
```
新物品得分：[0.06562606, 0.06562608, 0.06562608, ...]（完全相同）
已知物品得分：[0.04..., 0.90, 0.95, 0.97]（有区分度）
新物品排名：64.5/100，MRR=0.017
已知物品排名：34.5/100（含负样本均值）
```

### 关键发现 3：验证集正样本 vs 测试集正样本分布不同

```
验证集（split=1）正样本：
  - 100% 来自训练集（split=0）中的已知物品
  - 0% 新物品
  
测试集候选分布：
  - 46.5% 已知物品（可能是正样本）
  - 53.5% 全新物品（可能是正样本）
```

→ **验证永远不测冷启动，而测试大量包含冷启动**

### 关键发现 4：时序断层

```
训练集（split=0）结束：2009-12-28
验证集（split=1）结束：2011-01-29  ← 模型看过但未充分学习
测试集开始：2011-01-30              ← 完全在 split=0 之外 348 天
```

目前模型用 split=0 训练，以 split=1 验证。但 **split=1（2010年的互动）** 包含大量对预测 2011 年行为更有价值的近期信息，而我们只把它当验证集用，没有训练。

### 关键发现 5：seq_len 严重不足

```
用户平均交互次数：714 次
当前 seq_len=32：仅使用最近 4.5% 的历史
建议 seq_len=128：使用最近 17.9% 的历史

Dataset2 用户极端偏斜：
  - 25%位 4 次交互（轻量用户）
  - 50%位 21 次查询
  - 75%位 83 次查询
  - max：3714 次查询
```

### 理论最优上限分析

假设真实正样本中有 P% 是已知物品：

| 场景 | 已知正样本 MRR | 新物品正样本 MRR | Dataset2 MRR |
|------|--------------|----------------|-------------|
| 当前（pairwise_acc=0.97）| 0.42 | 0.017 | ~0.20 |
| 改进（pairwise_acc=0.99）| 0.69 | 0.017 | ~0.33 |
| 极致（pairwise_acc=0.999）| 0.95 | 0.037 | ~0.46 |

**结论：要达到 0.43 MRR，需要 pairwise 准确率从 0.97 提升到 0.999**

## 改进方案（优先级排序）

### 方案1：训练全部数据（split=0 + split=1）[最重要，最易实现]

**原理：** 当前只用 split=0（1.99M 交互，截至 2009-12）训练。把 split=1（266k 交互，2010年-2011年）也加入训练，给模型更多近期信息。

**收益：**
- 更多训练数据（+13%）
- 最重要：模型学到 2010-2011 年的行为模式，时间上更接近 2011-2012 测试期

**实现：** 修改数据加载，把 split=1 也作为 training edges 而不是 validation edges

### 方案2：增大 seq_len [重要]

**原理：** seq_len=32 → 64 或 128，更好地捕捉用户的长期兴趣

**收益：** 对每位用户，模型看到更多历史，尤其对重量级用户（>100 次交互）有帮助

**代价：** 计算成本约 2x

### 方案3：冷启动物品处理 [理论上很重要，实现复杂]

**原理：** 新物品目前得分 = 0.065（几乎相同），需要给它们差异化得分

**可能的信号：**
- 物品 ID 的相对大小（较新的 ID 可能代表更新的物品）
- 在测试集中作为候选出现的频率（测试中出现 100+ 次的物品更可能是热门）
- 但分析显示：所有新物品平均出现 137 次（均匀），无法区分

**当前结论：** 除非有外部物品特征（类型、标签等），否则无法有效区分新物品

### 方案4：100-way 评估验证 [修复指标误导]

**原理：** 用 99 个负样本评估（模拟测试），而不是 1 个随机负样本

**收益：** Val 指标才能真实反映 Test 性能，避免虚假繁荣

### 方案5：更难的负样本训练 [重要]

**原理：** 测试集负样本不是随机的，而是竞争性的（用户可能考虑过但没选的物品）。训练用更难的负样本，模型学会更细粒度的区分。

**实现：** 用 In-batch 负样本（其他用户的正样本作为负样本）或流行度加权负样本
